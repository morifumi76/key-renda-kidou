# ⌨️ key-renda-kidou 要件定義書

> キー連打でアクションを起動するmacOS常駐ランチャー
> 対象キーを2回もしくは3回すばやく連打（ノック）すると、割り当てたアクション（URLを開く／アプリ起動）を実行する。自分専用アプリ。参考アプリ：[Keyknockr](https://genjiapp.com/mac/keyknockr/)
>
> 出典: Notion「キー連打起動」(2026-07-15)

## 1. 概要

| 項目 | 内容 |
| --- | --- |
| アプリ名 | **key-renda-kidou** |
| 種類 | macOS用の常駐ランチャーアプリ（メニューバー常駐） |
| 利用者 | 自分専用（配布・署名・公証は不要） |
| 動作環境 | MacBook Air M4／現行macOS／JIS配列キーボード |
| 想定技術 | Swift + SwiftUI（実装詳細はClaude Codeに委任） |
| 差別化ポイント | Keyknockrは修飾キーのみ。本アプリは英数・かなキー、左右⌘⇧まで対象を拡張し、設定UIの分かりやすさを最重視する |

## 2. 基本動作フロー

```
[常駐]                [トリガー]              [実行]
メニューバーに          対象キーを連打            割り当てたアクション実行
アイコン表示    ──▶    （2回 or 3回）    ──▶    ├ URLを開く（デフォルトブラウザ）
 │                                             └ アプリを起動（起動中ならアクティブ化）
 └ オン/オフ切り替え・設定画面・終了
```

## 3. 対象キー（9キー・最大9登録）

| # | キー | 左右区別 | 注意事項（設定画面のツールチップで案内） |
| --- | --- | --- | --- |
| 1 | ⌘ Command（左） | ✅ | — |
| 2 | ⌘ Command（右） | ✅ | — |
| 3 | ⇧ Shift（左） | ✅ | — |
| 4 | ⇧ Shift（右） | ✅ | — |
| 5 | ⌃ Control | —（左のみ存在） | — |
| 6 | ⌥ Option | —（左のみ存在） | — |
| 7 | fn 🌐（地球儀） | — | macOS標準の機能（絵文字表示等）と競合するため、システム設定 → キーボード →「🌐キーを押して」を**「何もしない」**に変更するよう促す |
| 8 | 英数 | — | 押すたびに入力モードが切り替わる副作用がある（連打後は英数モードになる）ことを案内 |
| 9 | かな | — | 同上（連打後はかなモードになる） |

### 対象外にしたキーと理由

| キー | 理由 |
| --- | --- |
| A〜Z、1〜0 | 通常のタイピングと衝突し、文字も入力されてしまう |
| Esc、F1〜F12 | 操作キャンセル・メディアキーとの競合を避けるため見送り |
| Space / Tab / Enter / Delete | 文字入力・改行・削除がそのまま発生する |
| 矢印キー | カーソル移動で日常的に連打するため誤発動が多発する |
| Caps Lock | 大文字ロックが切り替わる副作用がある |

## 4. トリガー仕様（ノック判定）

- 各キーには **「2回押し」または「3回押し」のどちらか一方のみ**を設定できる（両方登録は不可）
  - これにより「3回目を待つ遅延」が発生せず、すべてのアクションが即発動する
- **ノックの定義：対象キーを単独で「押して離す」こと**

> 🚨 **最重要仕様：ショートカット操作との区別**
>
> - ⌘C → ⌘V のように**他のキーと組み合わせて押した場合は、ノックのカウントをリセットする**
> - **長押し**（一定時間以上の押しっぱなし）はノックとしてカウントしない
> - この仕様がないと、コピペのたびに誤発動する。本アプリの品質を決める心臓部として最優先で正しく実装すること

- ノックと判定する間隔：デフォルト **0.3秒**。設定画面のスライダーで **0.2〜0.5秒** の範囲で調整可能

## 5. アクション仕様

| アクション | 内容 | 入力チェック |
| --- | --- | --- |
| URLを開く | 指定URLをデフォルトブラウザで開く | URL形式のバリデーションを行う |
| アプリを起動 | 指定アプリを起動。すでに起動中ならアクティブ化 | 手打ちさせず、アプリ一覧やFinderからの選択式にする |

- 発動時のフィードバック：アクション実行時にメニューバーアイコンが一瞬変化する（「押せたか分からない」を防ぐ）

## 6. 設定画面（UI最重視）

> 🎨 **このアプリの肝は「設定のやりやすさ」。** キーボードの見た目そのままの設定画面を採用する。

```
┌─────────────────────────────────────────────┐
│  キーボード見た目UI（実際のJIS配列どおりに9キーを配置）              │
│                                                             │
│  [⇧左]                                    [⇧右]            │
│  [fn][⌃][⌥][⌘左][英数][ スペース ][かな][⌘右]                │
│                                                             │
│   ● 色付き＝登録済み  ○ 灰＝未登録（登録状況がひと目でわかる）      │
├─────────────────────────────────────────────┤
│  選択中： ⌘ Command（右）                                     │
│  ┌─────────────────────────────────────┐   │
│  │ トリガー： ◉ 2回押し ／ ○ 3回押し（択一）                │   │
│  │ アクション： [URLを開く ▼] https://www.yahoo.co.jp   │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

- キーをクリック → 下部にそのキーの設定パネル（トリガー択一＋アクション）を表示
- 登録状況を色で表示し、空きスロットが一覧できる
- **一覧表ビュー**（全9キーのリスト表示）も切り替えで用意する
- fn／英数／かなキーには注意事項のツールチップを表示する（3章参照）

## 7. メニューバー・常駐機能

- メニューバーメニューの構成：**「有効/無効の切り替え」「設定を開く」「終了」**
- ログイン時自動起動：設定でON/OFF可能
- 設定の保存：設定ファイル1個（JSON等）に保存。**「設定リセット」ボタン**も用意する

## 8. 権限まわり（初回起動オンボーディング）

- キー監視には macOS の**「入力監視」または「アクセシビリティ」の許可**が必要
- 初回起動時に「許可が必要 → システム設定を開く」ボタン付きの案内画面を表示する
- **許可がない状態を検知して、メニューバーアイコンで警告表示**する

## 9. テスト観点

- [x] ⌘C→⌘V を連発しても誤発動しないか（最重要）
- [x] 対象キーの長押しで誤発動しないか
- [x] 日本語入力中にノックした場合の挙動は問題ないか
- [x] スリープ復帰後もキー監視が生きているか
- [x] 権限を取り消した状態で警告表示されるか
- [x] 2回押し設定のキーが遅延なく即発動するか

※ ノック判定ロジックは `swift run knock-tests` で自動テスト済み。実機での動作確認も別途行うこと。
