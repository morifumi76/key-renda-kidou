# ⌨️ key-renda-kidou

キー連打（ノック）でアクションを起動する、macOS用のメニューバー常駐ランチャーアプリ。

対象キーを2回または3回すばやく連打すると、割り当てたアクション（URLを開く／アプリを起動）を実行します。

## 特徴

- **9つの対象キー**: 左⌘・右⌘・左⇧・右⇧・⌃・⌥・fn🌐・英数・かな
- **誤発動しない**: ⌘C→⌘Vのようなショートカット操作や長押しはノックとして数えません
- **キーボード見た目の設定画面**: JIS配列そのままのUIでキーをクリックして設定
- **即発動**: 各キーは「2回押し」か「3回押し」のどちらか一方だけを登録できる仕様のため、判定待ちの遅延がありません

## 動作環境

- macOS 14以降（Apple Silicon / JIS配列キーボード想定）
- ビルドにXcodeは不要（コマンドラインツールのみでOK）

## ビルドとインストール

```bash
# ビルドして ~/Applications にインストール＆起動
scripts/build-app.sh --install

# ビルドのみ（build/key-renda-kidou.app が作られる）
scripts/build-app.sh
```

## 初回起動時の設定（重要）

キー監視のために macOS の「入力監視」権限が必要です。

1. 初回起動時に表示される案内から「システム設定を開く」をクリック
2. システム設定 → プライバシーとセキュリティ → **入力監視** で `key-renda-kidou` をオン
3. 反映されない場合はアプリを再起動

また、**fn🌐キー**を使う場合は、システム設定 → キーボード →「🌐キーを押して」を**「何もしない」**に変更してください。

> **再ビルド時の注意**: コードを更新して再ビルドすると署名が変わるため、「入力監視」の許可を一度オフ→オンし直す必要がある場合があります。

## 使い方

1. メニューバーの ⌨️ アイコン →「設定を開く…」
2. キーボードUIで設定したいキーをクリック
3. トリガー（2回押し／3回押し）とアクション（URL／アプリ）を設定
4. 対象キーを連打するとアクションが発動（メニューバーアイコンが一瞬 ⚡ に変わります）

## テスト

ノック判定エンジン（誤発動防止ロジック）のテストを実行できます：

```bash
swift run knock-tests
```

## 設定ファイル

`~/Library/Application Support/key-renda-kidou/config.json` に保存されます。
設定画面の「設定をリセット」ボタンで初期化できます。

## プロジェクト構成

```
└── KnockTests/          # テストランナー
scripts/
├── build-app.sh         # .appバンドルのビルドスクリプト
└── Info.plist           # アプリバンドル用の設定ファイル
docs/specs/              # 要件定義書
```
